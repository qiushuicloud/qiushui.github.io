使用Logstash将Logback日志记录到MongoDB

发表于 2018-08-31 | 分类于 后端 ， ELK |

​ 在使用微服务架构时，将应用的日志以文件的形式记录在分散的机器上，不方便错误日志的查询。如果能把日志文件统一输出到某一个地方，集中存储那就再好不过了。

使用ELK（Elasticsearch + Logstash + Kibana）架构是解决分布式日志存储是最优的选择。然而有的时候并不想去使用Elasticsearch怎么办。好在Logstash提供了各种插件，可以将数据传输到多种平台。

什么是Logstash？

Logstash是elastic公司旗下的一个产品，ELK三大件之一。它是一个数据流的管道，可以理解为将数据从一个地方传输到另一个地方的工具。使用Logstash的场景：

  • 数据同步，比如将mysql的数据同步到Elasticsearch
  • 日志收集
  • 触发消息队列

官网地址：https://www.elastic.co/products/logstash

Logstash的结构

logstash分为三块结构：

  • input 数据输入块
  • filter 数据处理块
  • output 数据输出块

logstash的插件都是这三大块的插件，根据不同的输入输出端平台有不同的插件。

安装Logstash

下载地址：https://www.elastic.co/downloads/logstash

根据自己所用的平台选择压缩包类型，解压即用。

运行 /bin/logstash 就能启动logstash
