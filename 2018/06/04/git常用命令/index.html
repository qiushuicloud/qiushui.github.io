git常用命令

发表于 2018-06-04 | 分类于 开发工具 |

首先需要理解git的几个概念：

  • 工作区（Working Directory） 是指硬盘上的某个文件夹，里面的文件需要被git版本控制
  • 版本库（Repository）指工作区内的.git隐藏文件
  • 暂存区（Stage）版本库里的一块区域，用于存放修改了但未提交的文件
  • 提交（Commit）就是把暂存区里的文件移交给分支的操作，每个提交会有一个唯一的提交码
  • 分支（Branch）记录了文件的历史提交，不同的分支体现文件不同的发展方向，git默认会生成一个master主分支
  • 标签（Tag）对某个里程碑式的提交做出的别名，方便以后提取这一次的文件状态

git起步

git init

初始化

在工作区初始化一个空的git版本库

git clone

克隆

已有远程仓库时，从远程仓库克隆资源

git 配置

git config

配置

常用参数：

  • --global 全局设定

常用命令：

  • git config user.name xxx 设置作者信息
  • git config user.email xxx@aa.com 设置作者邮箱
  • git config --global core.autocrlf true 设置换行符自动替换，windows一般用true， Linux/Mac一般用input

git remote

设置远程库

针对当前变化的操作

git add

添加

把文件添加到暂存区，可以理解为加入到下次提交的内容中

git rm

删除

git reset

回退

常用的参数：

  • --mixed git reset的默认参数，回退到指定版本文件还未加入到暂存区的阶段，文件内容还是最新状态，需要add再commit
  • --soft 回退到指定版本，只回退了提交，文件内容还是最新状态，可以再次commit
  • --hard 强行回退到指定版本，不仅回退了提交，连文件内容也回退了（慎用）

git reset HEAD^ 回退到上一个提交的状态

HEAD^ 表示上一个提交

HEAD^^ 表示上上一个提交

…以此类推

HEAD^n 表示第n个提交

git mv

移动/重命名

这个命令其实和Linux的mv类似

检查历史和状态

git log

检查历史

查看提交的历史记录

常用参数：

  • --oneline 提交信息显示在一行上
  • -2 显示最新2条
  • –skip=2 跳过2条
  • –graph 显示分支的发展路线，方便用于看合并
  • –decorate 显示HEAD 分支名，标签名

git show

查看

查看某次提交的详细信息和文件变更内容

git status

查看工作区状态

成长、标记和调整历史

git branch

分支

git tag

标签

git checkout

检出

git commit

提交

常用参数：

  • -a 提交前把所有修改的文件加入暂存区
  • -m 加入备注文字

git diff

查看差异

git merge

合并分支

远程库相关

git fetch

提取

从远程库上提取新的文件到版本库

git pull

拉取

从远程库上提取新的文件到版本库，并合并到工作区的文件中，git pull 等价于 git fetch + git merge

git push

推送

常用的一个命令是git push -u origin master 其中-u表示默认使用后面的远程库和分支名，下次推送只需要git push

其他常用推送：

  • git push --tags xxx 推送标签xxx到远程库
  • git push -d xxx 推送删除远程库的标签xxx
