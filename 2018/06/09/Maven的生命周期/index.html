Maven的生命周期

发表于 2018-06-09 | 分类于 后端 ， Maven |

Maven拥有三套独立的生命周期，分别是：

  • clean 清理项目
  • default 构建项目
  • site 建立项目站点

每个生命周期都包含不同的阶段（phase），并且阶段是有顺序的，后面的阶段依赖于前面阶段的执行，当调用后面的某个阶段时，排在前面的阶段也会被依次执行。三个周期的阶段互不影响。

clean生命周期

目的是清理项目，具体的操作是删除${project.build.directory} 默认值是target的文件夹

clean生命周期包含三个阶段：

阶段 说明
pre-clean 执行一些清理前需要完成的操作
clean 执行清理上一次构建生成的文件
post-clean 执行一些清理后需要完成的操作

default生命周期

定义了构建时需要执行的所有步骤

default生命周期包含的所有阶段：

阶段 说明
validate 验证阶段
initialize 初始化构建状态，例如设置属性或创建目录。
generate-sources 生成项目主源代码文件
process-sources 处理项目主源代码文件，例如替换值
generate-resources 生成项目主资源文件
process-resources 处理项目主资源文件，例如替换值
compile 编译项目主源码
process-classes 处理编译好的文件
generate-test-sources 生成项目测试源代码文件
process-test-sources 处理项目测试源代码文件
generate-test-resources 生成项目测试资源文件
process-test-resources 处理项目测试资源文件
test-compile 编译项目的测试代码
process-test-classes 处理编译好的测试文件
test 使用单元测试框架运行测试
prepare-package 打包前的准备操作
package 接受编译好的字节码， 打包成可发布格式
pre-integration-test 执行集成测试之前需要执行的操作
integration-test 执行集成测试
post-integration-test 执行集成测试之后需要执行的操作
verify 运行任何检查以验证包是否有效
install 将包安装到Maven本地仓库
deploy 将包部署到Maven远程仓库

site生命周期

目的是建立和发布项目站点

site生命周期包含四个阶段：

阶段 说明
pre-site 在项目站点生成之前需要执行的操作
site 生成项目站点文件
post-site 在项目站点生成之后需要执行的操作
site-deploy 将生成的站点文档部署到指定的web服务器
