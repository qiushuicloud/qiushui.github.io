从Java8升级到Java11的各种坑

发表于 2019-02-19 | 分类于 后端 ， Java |

耗时一天，翻阅了无数的Stackoverflow帖子和官方文档，终于解决了以下各种问题，供大家参考

Lombok的兼容Bug

这个是从Java9开始就出现的老问题。在没有使用Java9带来的Jigsaw 模块化新特性时，是可以正常使用Lombok的。一旦你添加了module-info.java 开启了具名模块，Lombok将不能正常工作。

报错各种找不到符号，以及以下提示：

或

参考链接

https://github.com/rzwitserloot/lombok/issues/1723

问题解决

module-info.java

将Lombok的版本升级到最新版本1.18.6

修改maven-compiler-plugin

重要提示：

我试了好长时间maven的编译阶段依然报错

最后发现maven-compiler-plugin的版本必须使用3.8.0才行，或者升级maven到3.6.0

Mapstruct不能生成Mapper接口

module-info.java

参考链接

https://github.com/mapstruct/mapstruct/issues/1543

问题解决

修改maven-compiler-plugin

使用spring-data-mongodb的错误

在spring-boot-starter-data-mongodb内引用了mongodb-driver和mongodb-driver-core两个jar，这两个都有org.mongodb这个包，从而会发生冲突

参考链接

https://jira.mongodb.org/browse/JAVA-2521

问题解决

把mongodb-driver和mongodb-driver-core排除，并且添加入mongo-java-driver

在运行spring-boot时会出现一个错误

这是因为自从Java9以后JDK 把jaxb的包都排除了，需要时自行添加

参考链接

https://stackoverflow.com/questions/43574426/how-to-resolve-java-lang-noclassdeffounderror-javax-xml-bind-jaxbexception-in-j

问题解决

xstream的警告

在使用spring-cloud时会出现这么一个警告

参考链接

https://github.com/x-stream/xstream/issues/101

问题解决

dockerfile-maven插件的问题

使用dockerfile-maven-plugin 打包docker镜像时会报错

参考链接

https://github.com/spotify/dockerfile-maven/issues/90

问题解决

还是由于从JDK9开始不再自带javax.activation包引起的

给插件添加activation包的依赖就解决了
